Atom折腾的备份

  • Atom

折腾这么长时间Atom了，把一些心得写下来，方便以后重新配置的时候，得心应手些。

前言

在折腾 Atom 的过程中，我搜集过很多资料，不可否认的是 Atom 社区的活跃度很高，大家都爱分享。可有一点真的让人很不爽，当我多次遇到某些细节上的问题时，搜索结果（中文）更多是重复性很高的二手文章，一遍遍筛选的过程中浪费了我很多时间。其实真的没必要将一些别人写过很多遍的东西自己再重复写一遍，这样没多大意义，所以下面我写的都主要是我在一遍遍使用过程中碰到的具体问题，如果能帮到你，那就太好了。

如果你刚入门，不建议你直接看，你可以在有一定基础的情况下看下面的内容。

主题

先说主题，毕竟我们都是看颜值的人。不过这个还是看个人喜好，其实自带的主题挺不错的。

我从一开始用的主题就是 MaterialDesign，毕竟我是 Android 开发者出身，怎么能不喜欢 Material Design 呢？

这是地址：

https://atom.io/themes/atom-material-ui
https://atom.io/themes/atom-material-syntax

快捷键

自带快捷键

我说几个使用频次比较高的

快捷键 作用
ctrl-shift-p 唤起 Atom 命令面板的
ctrl-， 打开Seeting界面的
ctrl-k+↑↓←→ 实现分屏
ctrl+n 新建文件
ctrl-alt+←→ 多标签来回切换

差不多就这么几个最常用了，想看更多的快捷键可以去这里
Atom常用快捷键

自定义快捷键

打开 setting 点击左侧 keybingings 这是你会看到一堆快捷键，看哪个不爽改哪个就行，比如我最常用的是删除行，搜索 delete-line 找到默认的，然后复制到 keymap.cson 里，修改成我想要的 ctrl-d 就OK了！其他同理。

插件

在我看来 Atom 最大的优点就是它的插件真的太多太爽了，其实最开始我也研究过 SublineText 但
是讲真，对于我这种比较愚钝的人来说，上手难度还是有点大。

下面我列举一下我所用到的插件，主要是做 Web 开发相关。

  • minpmap 实用性？不知道，反正右侧有一堆代码的缩略感觉很酷的样子。
  • atom-beautify格式化代码的，默认快捷键是 ctrl-alt-b。
  • emmet 用过之后就离不开的那种，懂得自然懂，不懂得自己去查，可以看看这个Emmet使用手册
  • file-icons 看看名字就知道了，可有可无，有了更好。
  • pretty-json格式化 json 的，快捷键不知道，如果你知道可以告诉我，怎么使用？请永远记住 ctrl-shift-p 然后输入插件名字，你会看到你想看到的，然后点击就行。
  • run-in-browser去浏览器查看，默认快捷键 ctrl + alt + r 不过很可能装了这么多插件，会让默认快捷键失效，不过可以自定义，往下看。
  • highlight-line顾名思义，高亮鼠标选中的那一行
  • highlight-selected高亮用户选中的词汇

一些小TIPS

Atom的自动保存

很长时间以来，Atom 最令我反感的就是每次写完代码后，还要像一个原始人一样 ctrl-s 保存一下代码。这还像一个现代化编辑器该有的问题吗？后来才发现，其实 Atom 自带了自动保存功能，原来一直都是我的无知造成的误会。去 Packages 中搜索 autosave 然后 setting 勾选上 Enabled 就行，不是很懂为什么不将此选项自动设置为默认。

自动上传FTP

首先声明一点，我也就是一个刚开始学习前端的新手，当我第一次知道有FTP的时候，觉得挺好玩，但是每次都要将写好的代码放到FTP里好麻烦，就在想有没有像Git一样的东西，可以在编辑器里上传？当然可以啊，答案就是 RemoteSync这个插件。
安装好之后，在你当前目录下右键会出现 RemoteSync 的选项，先配置好你的账号之类的东西，然后就可以 Upload 了
